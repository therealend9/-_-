# 考试文件识别模块安全问题补充评估与政策依据

版本：v1.0  
评估日期：2026-07-19  
适用范围：试题卷识别、空白答题卡模板识别、学生答题卡识别、结果脱敏与结果交付

## 1. 结论摘要

当前版本已经具备以下安全基础：

- 在最终结果持久化和 API 返回前执行规则型数据脱敏；
- 脱敏审计只保存规则版本、命中类别、数量和哈希，不保存原始 OCR 文本；
- 脱敏或审计失败时阻止结果落盘和成功返回；
- 固定答题卡的姓名、学号区域可以与答题区域隔离；
- 模板内部路径不会作为对外响应字段返回；
- 输入文件已有扩展名、MIME、文件签名、尺寸、页数和 DOCX 压缩包等限制。

但这还不能称为完整的生产安全闭环。当前最重要的剩余问题不是再增加几条正则，而是：

1. API 和静态资源缺少生产级认证、授权和对象级权限校验；
2. 原始文件、中间文件和结果文件缺少明确的加密、留存、删除和恢复策略；
3. 访问、下载、人工复核、导出和交付行为没有形成完整审计链；
4. 尚未落实安全事件响应、个人信息保护影响评估和定期合规审计；
5. OCR 运行环境、模型文件、依赖包和外部模型边界尚未形成供应链与网络出口控制；
6. 规则型脱敏尚未通过真实授权样本完成漏检率、误杀率和低置信 OCR 评估。

本模块仍只负责扫描、切分、识别、结果脱敏和安全交付，不负责评分、批改建议或最终成绩决定。

## 2. 政策与规范依据

下列资料按照“直接法律依据”“部门规章/行政规范”“工程和教育治理参考”区分。国际资料和工程框架不能替代中国法律要求。

### 2.1 直接法律依据

#### 《中华人民共和国个人信息保护法》

官方来源：中国政府网  
https://www.gov.cn/xinwen/2021-08/21/content_5632486.htm

与本模块直接相关的要求：

- 第四至第九条：处理个人信息应具有合法、正当、必要、明确的目的，遵循最小化和诚信原则；
- 第二十八至第三十二条：敏感个人信息应具有特定目的和充分必要性，并采取更严格保护措施；
- 第三十八至第四十条：向境外提供个人信息需要满足法定条件并履行相应义务；
- 第五十一条：应根据数据种类、数量、目的和方式采取安全措施，防止未经授权访问、泄露、篡改和丢失；
- 第五十五至第五十六条：处理敏感个人信息、委托处理、向境外提供等活动，应在适用情形下开展个人信息保护影响评估并留存记录；
- 第五十七条：发生泄露、篡改或丢失时，应采取补救措施并按要求通知；
- 第五十九条：受托处理者应按约定处理个人信息并采取必要安全措施。

对本模块的落地含义：姓名、学号和答题内容应按不同权限域处理；脱敏必须与访问控制、留存删除、事件响应和委托方管理同时成立，不能把“输出中没有敏感字符串”当作全部合规措施。

#### 《中华人民共和国数据安全法》

官方来源：中国政府网  
https://www.gov.cn/xinwen/2021-06/11/content_5616915.htm

重点要求：建立数据分类分级保护制度，采取必要措施保障数据安全，建立数据安全风险监测、预警和应急处置机制，并承担数据处理活动的安全责任。

对本模块的落地含义：应把原始答题卡、身份区域、OCR 中间结果、脱敏结果、模板和未公开试题分级管理，分别规定访问角色、处理目的、留存期限、删除方式和异常处置责任。

#### 《中华人民共和国网络安全法》

官方来源：中国政府网  
https://www.gov.cn/xinwen/2016-11/07/content_5129723.htm

说明：该链接是法律公布页面。正式上线审查时必须以全国人大常委会公布的最新现行文本、修正案和配套规定为准，不应仅依赖历史页面或项目内摘录。

重点要求：网络运营者应履行网络安全保护义务，采取技术措施防止网络数据泄露、损毁和丢失，并保障日志留存、运行安全和事件处置能力。

对本模块的落地含义：生产部署必须使用 HTTPS、身份认证、最小权限、访问日志、漏洞修复、备份恢复和网络隔离；识别 Worker 不应直接暴露存储目录或具备不必要的外网访问能力。

#### 《网络数据安全管理条例》

官方来源：中国政府网政策文件库  
https://www.gov.cn/zhengce/content/202409/content_6977766.htm

重点关注：网络数据处理者应建立数据安全管理制度，采取加密、备份、访问控制和安全认证等措施，规范委托处理、重要数据保护和安全事件处置。

对本模块的落地含义：应建立数据流清单，明确原始文件、识别结果、脱敏结果和备份的存储地域、责任主体、访问方式和删除状态；禁止使用未知地域、未知留存策略的外部对象存储或模型接口。

### 2.2 部门规章与行政规范

#### 《生成式人工智能服务管理暂行办法》

官方来源：中国网信网  
https://www.cac.gov.cn/2023-07/13/c_1690898327029107.htm

重点要求：训练数据应有合法来源；涉及个人信息时应取得同意或具有其他法定依据；保护输入信息和使用记录，不得收集非必要个人信息，不得非法留存能够识别身份的信息。

对本模块的落地含义：当前识别模块默认本地 OCR；未来如调用外部模型，必须先完成字段白名单、脱敏、供应商留存/训练/共享政策审查和跨境评估。`llm_enabled` 不能成为绕过脱敏边界的开关。

注意：该办法主要针对向境内公众提供生成式人工智能服务的场景。当前识别模块是否直接落入其服务提供者范围，应结合部署形态判断；但个人信息保护、数据安全和网络安全要求仍然适用。

#### 《个人信息保护合规审计管理办法》

官方来源：中国网信网  
https://www.cac.gov.cn/2025-02/14/c_1741233507681519.htm

施行日期：2025-05-01。

重点要求：个人信息处理者应建立个人信息保护合规审计制度，围绕处理规则、最小化、安全措施、委托处理、跨境提供、权利保障和事件处置开展审计，并对发现的问题整改留痕。达到规定规模的处理者还需要按照规定周期开展定期审计。

对本模块的落地含义：当前“脱敏审计 JSON”只是单次处理证据，不等同于个人信息保护合规审计。还需要建设周期审计计划、审计范围、问题单、整改责任人、复测结果和证据归档。

#### 《人工智能生成合成内容标识办法》

官方来源：中国网信网  
https://www.cac.gov.cn/2025-03/14/c_1743654684782215.htm

与当前识别模块关系较弱，主要影响下游批改建议、报告或其他生成内容的标识、传播和平台责任。本模块不生成批改建议，因此不应把该办法当作本模块的核心整改项；但需要在接口交付边界中明确下游责任归属。

## 3. 剩余安全问题分级

### P0：上线前必须闭环

| 编号 | 问题 | 当前状态 | 必须达到的闭环 |
|---|---|---|---|
| P0-01 | API 未完成认证、授权和对象级访问控制 | 代码可调用接口，但未形成生产身份体系 | 登录身份、角色、考试范围和 `submission_id` 逐项校验；无权限统一拒绝 |
| P0-02 | 静态模板资源可能绕过接口权限 | `/api/template-assets` 存在直接静态挂载风险 | 改为鉴权下载或私有存储签名地址；禁止目录遍历和未授权原图访问 |
| P0-03 | 原始文件、身份区域和中间 OCR 结果的保护边界不完整 | 有目录划分和上传隔离设计，尚无完整生产存储安全 | 私有存储、服务账号隔离、磁盘/对象存储加密、密钥管理、备份保护和访问审计 |
| P0-04 | 脱敏失败不能保证所有输出路径都阻断 | 最终结果路径已有控制 | 对 API、查询接口、题目目录、缓存、调试输出和异步任务统一做交付前检查 |
| P0-05 | 处理结果和原始文件缺少正式删除闭环 | 有 quarantine 清理设计，其他目录留存期限尚未统一 | 按数据类型配置 TTL，删除原始文件、页面、中间结果、缓存和备份；记录删除证据并支持恢复演练 |

### P1：生产环境必须补齐

| 编号 | 问题 | 风险 | 建议措施 |
|---|---|---|---|
| P1-01 | 异常信息可能返回 `str(exc)` | 泄露路径、解析器信息和内部实现 | 对外返回固定错误码和关联 ID，详细原因只写受控日志 |
| P1-02 | 访问审计不完整 | 无法确认谁查看、下载、导出或复核了学生数据 | 记录查看、下载、模板发布、人工复核、导出、交付、删除和权限变更事件 |
| P1-03 | 没有数据处理清单和 PIA | 无法证明目的限定、必要性和留存期限 | 建立字段级数据流登记和个人信息保护影响评估，变更时重新评估 |
| P1-04 | 外部模型和供应商边界未落地 | 输入留存、训练使用、第三方共享和跨境传输不可控 | 供应商准入、合同约束、零训练/留存政策确认、字段白名单、出口域名白名单和地域登记 |
| P1-05 | 模型与依赖供应链缺少可验证台账 | 恶意模型、依赖漏洞或版本漂移 | 固定模型版本和 SHA-256，生成 SBOM，漏洞扫描，生产禁止自动下载和未经审批升级 |
| P1-06 | 未公开试题和答题内容缺少数据分级 | 试题泄露影响考试公平，答案泄露影响学生隐私 | 试题、原始答题卡、身份区域、脱敏答案分别分级，采用最小权限和用途绑定 |
| P1-07 | 低置信 OCR 仍可能静默交付 | 脱敏漏检、题号误识别和错误绑定 | 以置信度、身份区域检测、异常字符和脱敏命中结果触发复核；未复核结果禁止交付 |
| P1-08 | 审计日志自身可能包含敏感信息 | 日志成为第二泄露渠道 | 日志字段白名单、正文禁止原文、路径脱敏、日志访问分权、传输加密和到期删除 |
| P1-09 | 缺少泄露事件应急闭环 | 发现泄露后无法快速止损和履责 | 建立分级、告警、阻断、证据保全、通知、恢复、复盘和演练流程 |
| P1-10 | 没有正式的合规审计制度 | 单次脱敏报告不能替代定期审计 | 每年至少进行一次内部审查；达到法规规定条件时按规定周期开展合规审计 |

### P2：持续优化

- 按用户、考试、IP、任务和 Worker 维度限流，设置并发、CPU、内存、磁盘和处理时长配额；
- 生产 CORS 仅允许受控前端来源，启用 HTTPS、HSTS 和安全响应头；
- 对模板版本、题目目录和结果建立签名或完整性校验，检测存储篡改；
- 对真实授权样本持续统计脱敏召回率、误杀率、低置信漏检率和身份区域裁剪漏出率；
- 保留模型、规则、模板和 Schema 版本，实现结果可回放和问题可定位；
- 对 OCR 文本执行控制字符、异常 Unicode、超长文本和提示词注入隔离；OCR 文本只能作为数据，不能作为系统指令。

## 4. 责任边界

### 4.1 识别模块必须负责

1. 安全接收文件，完成格式真实性和资源消耗校验；
2. 试题卷自由格式识别和答题卡模板/答题区切分；
3. 在结果交付前执行规则型数据脱敏；
4. 不向下游暴露原始文件路径、身份映射和未脱敏中间结果；
5. 对脱敏失败、低置信结果和异常输入阻断交付；
6. 输出结构继续保持 `exam-document.v1`，不改变既有输入输出结构；
7. 提供处理、脱敏、失败和交付的最小审计证据。

### 4.2 部署平台必须负责

- 统一认证、角色权限、考试范围和对象级授权；
- 私有存储、加密、密钥、备份、留存删除、日志平台和网络隔离；
- PIA、数据处理清单、委托处理协议、跨境评估和合规审计；
- 事件响应、漏洞管理、供应链管理、HTTPS、限流和生产运维。

### 4.3 批改或外部模型模块负责

- 是否调用外部模型及其供应商合规审查；
- 评分建议、批改结果、人工复核、申诉和成绩决定；
- 生成内容的标识、使用记录和下游访问权限。

## 5. 建议的安全闭环

```text
认证与考试授权
  -> 安全上传与隔离保存
  -> 文件真实性/恶意内容/资源消耗校验
  -> 受控 OCR 与模板切分
  -> 身份区域与答题区域空间隔离
  -> 结果级脱敏与 Schema 校验
  -> 低置信/异常/脱敏失败阻断或人工复核
  -> 只交付脱敏后的 exam-document.v1
  -> 访问、下载、复核、导出、交付审计
  -> 按留存策略删除并保留删除证据
  -> 定期测量、合规审计、事件演练和规则迭代
```

任何环节失败都不能返回“处理成功”。原始文件和身份区域只在受控内部权限域可见，交付给下游的关联键只能使用 `submission_id` 和 `question_id`；本模块不把姓名、学号或身份映射交给批改模块。

## 6. 推荐实施顺序

### 第一阶段：阻断高风险暴露

1. 在生产反向代理和 API 中接入认证、角色权限和 `submission_id` 对象授权；
2. 移除模板目录静态公开访问，统一改为鉴权资源读取；
3. 统一异常响应，禁止向客户端返回原始异常文本和服务器路径；
4. 关闭生产 `include_intermediate`，禁止调试中间结果外发；
5. 明确原始文件、页面、中间结果、结果和日志的存储权限。

### 第二阶段：建立数据生命周期

1. 完成考试文件数据处理清单和 PIA；
2. 为每类文件定义留存期限、删除方式、备份同步和恢复权限；
3. 建立访问、下载、导出、复核、交付和删除审计；
4. 建立泄露事件响应预案并进行一次演练；
5. 完成 OCR 模型、依赖和外部服务供应商台账。

### 第三阶段：验证脱敏实际效果

1. 使用授权的合成样本和公开授权样本建立覆盖集；
2. 测试姓名、学号、手机号、邮箱、身份证、文件名、路径、控制字符和 OCR 错误变体；
3. 测量脱敏召回率、误杀率、区域裁剪漏出率和低置信阻断率；
4. 对每次规则、模型、模板和 Schema 变更执行回归测试；
5. 由数据保护负责人或学校信息化部门对上线标准签字确认。

## 7. 当前版本验收标准

- 未认证或无考试权限的调用无法上传、查询、下载或查看模板原图；
- 任意输出路径都不会返回未脱敏的姓名、学号、手机号、邮箱、身份证号、绝对路径或原始文件名；
- 脱敏失败、审计失败、Schema 失败、低置信或身份区域异常不会静默交付；
- 原始文件、身份区域、OCR 中间结果和脱敏结果具有不同的存储权限；
- 每一次查看、下载、复核、导出、交付和删除都可按操作者、对象、时间和结果追溯；
- 超过留存期限的数据会被删除，删除和备份清理有证据；
- 生产环境不自动下载模型和依赖，不把未经批准的 OCR 文本发送到外部模型；
- `/api/process` 输入和 `exam-document.v1` 输出结构保持不变；
- 文档明确声明：规则型脱敏不是对任意手写姓名、学号和上下文隐私的绝对识别保证，必须结合授权样本测试和人工复核策略。
