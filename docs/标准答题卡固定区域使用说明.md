# 标准答题卡固定区域使用说明

## 模板与考试绑定

模板保存在 `storage/templates/<template_id>/template.json`，区域坐标必须是
相对于空白答题卡页宽高的归一化坐标。创建模板前，必须先用同一 `exam_id` 处理试题卷。
试题卷输出的 `questions[]` 会保存为考试题目目录，目录中的 `question_id` 是模板和最终
答案的唯一关联键。手工创建模板时也必须使用目录中的 ID，不能根据答题卡印刷题号重新编号：

```python
from template_registry.service import bind_exam_template, publish_template, save_template

save_template({
    "template_id": "tpl_mayuan_2026_v1",
    "version": 1,
    "pages": [
        {
            "page_no": 1,
            "reference_width": 2480,
            "reference_height": 3508,
            "reference_image_path": "page_001.png",
            "regions": [
                {
                    "question_id": "2.1",
                    "question_no": "2.1",
                    "template_label": "1",
                    "order": 1,
                    "bbox": [0.08, 0.16, 0.92, 0.30],
                    "coordinate_type": "normalized",
                    "ocr_mode": "handwriting"
                }
            ]
        }
    ]
})
publish_template("tpl_mayuan_2026_v1")
bind_exam_template("exam_2026_mayuan_01", "tpl_mayuan_2026_v1")
```

`reference_image_path` 相对于模板目录。考试发布后应新建模板版本，不能修改
已经绑定到考试的模板。

## 自动生成与人工复核

空白答题卡可以先由系统生成候选模板：

```powershell
python scripts/create_answer_sheet_template.py "blank_answer_sheet.pdf" `
  --template-id tpl_mayuan_2026_v1 `
  --template-name "2026 马原答题卡" `
  --version 1 `
  --exam-id exam_2026_mayuan_01
```

系统依据印刷题号和纵向位置推断答题区域，生成 `pending_review` 草稿。每个候选
区域默认写入 `needs_review: true`、`region_source: auto_label_layout`；前端应展示
候选框，允许教师拖动、删除、新增和确认。草稿不能绑定考试，只有复核后调用
`publish_template(template_id)` 发布，才可执行 `bind_exam_template(...)`。
自动映射后，候选区域保留印刷题号为 `template_label`，并使用题目目录中的
`question_id`、`question_no`。区域数量与题目目录不一致时，会标记
`question_mapping_error: "question_count_mismatch"`，必须人工复核后才能发布。

## 学生答题卷处理

学生提交只携带 `exam_id` 和答题卡文件。入口由服务端查询绑定模板，强制把 PDF
渲染为图像并走固定区域路径：

```powershell
python scripts/run_full_processor.py "answer_sheet.pdf" `
  --exam-id exam_2026_mayuan_01 `
  --document-role answer_sheet `
  --out outputs\answer_sheet_result.json
```

输出是紧凑的题目级答案记录：

```json
{
  "schema_version": "exam-document.v1",
  "submission_id": "sub_001",
  "exam_id": "exam_2026_mayuan_01",
  "template_id": "tpl_mayuan_2026_v1",
  "template_version": 1,
  "document_role": "answer_sheet",
  "layout_type": "fixed",
  "answers": []
}
```

模板区域增加 `content_type`。`question` 区域使用印刷体 OCR 并输出 `questions`；
`answer` 区域使用手写 OCR 并输出 `answers`。同一个 `question_id` 的多个区域按 `order` 合并。空白区域会输出空文本和
`is_blank: true`；页面对齐、裁剪或 OCR 质量异常会写入 `needs_review` 和
`risk_flags`。

## 依赖说明

安装 OpenCV 后，页面对齐优先使用 ORB/RANSAC。当前运行环境没有 OpenCV 时，
系统会按目标尺寸缩放后继续处理，并将结果标记为人工复核。生产环境应安装
`opencv-contrib-python`，并配置适用于手写中文的 OCR 引擎。
