# 答题卡学生信息区域隔离改造方案

## 1. 目标

在空白答题卡模板复核页中，允许教师创建、拖拽、缩放、删除和确认两类学生信息区域：

- `student_name`：姓名书写区。
- `student_no`：学号书写区。

身份区域只用于后续校内身份绑定能力的预留和空间隔离。本次不做姓名 OCR、学号 OCR、学生名单匹配或身份绑定；身份区域不会进入 `answers[]`、不会生成 `question_id`、不会裁剪为答题内容、不会发送给下游。

## 2. 兼容约束

以下接口和输出不变：

- `POST /api/process` 的全部输入字段不变。
- `GET /api/submissions/{submission_id}` 不变。
- `exam-document.v1` 不变。
- `answers[]`、`questions[]`、`question_id`、`question_no`、模板绑定逻辑不变。
- 模板复核接口仍只提交 `pages`；学生信息块直接作为既有 `pages[].regions[]` 中的区域对象，不增加新的请求体字段。

模板管理对象新增的仅是内部元数据：

```json
{
  "identity_protection": {
    "required": true,
    "status": "pending_review"
  }
}
```

历史模板没有该字段时视为 `legacy_unconfigured`：仍可运行，避免阻断已经发布或绑定的考试；但不具备“姓名、学号空间隔离已确认”的安全声明。新建草稿默认要求身份区复核。

## 3. 模板区域模型

答题区保留原模型：

```json
{
  "content_type": "answer",
  "question_id": "2.1",
  "question_no": "2.1",
  "order": 1,
  "bbox": [0.08, 0.20, 0.92, 0.45]
}
```

身份区使用同一坐标与编辑能力，但不关联题目：

```json
{
  "content_type": "student_name",
  "bbox": [0.12, 0.05, 0.42, 0.10],
  "coordinate_type": "normalized",
  "needs_review": false,
  "region_source": "auto_identity_label"
}
```

`student_no` 的格式相同。身份区不允许填写 `question_id`、`question_no` 或 `order`；后端在保存时会移除这些无效关联字段。

## 4. 生成与复核流程

```text
上传空白答题卡
  -> OCR 查找印刷“姓名”“学号”标签
  -> 在标签右侧生成候选身份块
  -> 与答题区候选块一并显示在同一复核画布
  -> 教师动态调整、删除或新增身份块
  -> 发布校验
  -> 冻结模板版本
```

对扫描版空白卡，自动标签定位可能不可用；教师仍可以手工新建两类身份块。自动候选永远带 `needs_review: true`，不能自动发布。

## 5. 发布校验

当模板的 `identity_protection.required = true` 时，发布必须满足：

1. 恰好一个 `student_name` 和一个 `student_no` 区域。
2. 两个身份区域均已复核，即 `needs_review` 为 `false`。
3. 身份区都在页面边界内。
4. 身份区之间不重叠。
5. 任一身份区不得与任一 `answer` 或 `question` 区域重叠。
6. 发布后模板不可变；调整身份区必须创建新的 `template_id` 和版本。

## 6. 学生答题卡运行链

```text
学生答题卡
  -> 根据 exam_id 取得已发布模板
  -> 页面配准
  -> 仅裁剪 content_type=answer 的区域
  -> 仅对答题区运行手写 OCR
  -> 输出 answers[]
```

固定答题卡路径不再对整页学生答题卡运行 OCR。姓名区、学号区可以参与页面配准的视觉图像处理，但不被裁剪、不被 OCR、不进入最终输出或内部中间 OCR 结果。这是空间隔离保障；最终规则型文本脱敏仍作为第二道防线。

## 7. 前端复核页实现要点

- 用现有区域框组件渲染三种 `content_type`，颜色和标签不同。
- 答题区显示题号并要求绑定 `question_id`。
- 姓名区、学号区不显示题号，不允许编辑题号字段。
- 自动候选块显示为待复核；教师确认后清除 `needs_review`。
- 发布按钮调用现有 `PUT /api/templates/{template_id}/review`，传回编辑后的 `pages`。
- 发布失败时展示后端的区域冲突、缺失或待复核错误，不自行忽略。

## 8. 验收与回滚

验收测试包括：自动候选、手工身份块、越界、身份区与答题区重叠、未复核阻断发布、历史模板兼容、固定答题卡只裁剪答题区、`/api/process` 输入和 `exam-document.v1` 输出回归。

回滚只需停止创建带 `identity_protection.required` 的新模板；历史模板和已有模板不迁移、不重写。若需要移除代码，模板中的身份块仍会被旧固定切分器忽略，因为旧逻辑只读取 `answer` 区域。
