# 识别模块前端改造方案

## 一、已接入的前端基线

共享包中的 Vue 3 + Vite + Element Plus 前端已复制到项目目录 `frontend/`。现有批改页面为：

- `frontend/src/views/teacher/GradingReview.vue`：教师端 AI 批改复核工作台。
- `frontend/src/views/student/GradingDetail.vue`：学生端批改结果详情。
- `frontend/src/views/admin/AssignmentDetail.vue`：作业详情和批改记录。

识别模块不替换批改负责人的页面和接口，只向批改工作台提供已经按题目切分的试题或学生答案。

## 二、建议的页面结构

识别模块不应只依赖批改复核页。完整前端至少需要以下独立页面：

```text
考试与任务管理页
  -> 试题卷识别结果页
  -> 空白答题卡模板审查页
  -> 学生答题卷识别结果页
  -> 批改复核工作台
```

其中“空白答题卡模板审查页”是独立页面，不是批改复核页中的一个普通按钮区域。批改复核页只消费已经确认的试题和学生答案结果。

### 1. 考试与识别任务页

用于选择考试并进入三类操作：上传试题卷、创建答题卡模板、处理学生答题卷。

按钮功能：

- `新建考试`：填写 `exam_id`、考试名称和考试状态，调用 `POST /api/exams`。
- `查看考试`：查看考试状态、题目目录、模板名称和模板版本。
- `上传试题卷`：选择文件和考试，调用 `POST /api/process`，固定传 `document_role=question_paper`。
- `创建答题卡模板`：上传空白答题卡，填写模板名称和版本，调用 `POST /api/templates/drafts`。
- `处理学生答题卷`：选择考试并上传学生答题卷，调用 `POST /api/process`，固定传 `document_role=answer_sheet` 和 `exam_id`。
- `查看处理结果`：根据 `submission_id` 调用 `GET /api/submissions/{submission_id}`。

### 2. 试题卷识别结果页

采用左侧原始页面、右侧题目列表的布局。点击题目时，在原图中高亮对应页码和题目区域。

按钮功能：

- `开始识别`：上传试题卷并显示处理状态。
- `上一页/下一页`：切换原始试卷页面。
- `定位题目`：跳转到题目所在页，并高亮其识别范围。
- `编辑题号`：修改显示题号；保存前校验题号不能重复。
- `编辑题干`：人工修正OCR文本，不修改原始图片。
- `标记复核`：给题目增加人工复核状态。
- `保存题目目录`：保存确认后的 `questions[]`，作为后续模板绑定的题目目录。
- `重新识别`：使用相同 `submission_id` 重试；已有成功结果时直接读取持久化结果，不重复执行。

### 3. 空白答题卡模板审查页

这是识别模块的核心页面。左侧显示答题卡原图，画布上叠加自动识别的答题区域；右侧显示当前区域属性。

按钮功能：

- `放大/缩小/适应页面`：只改变画布显示比例，不改变模板坐标。
- `选择区域`：选中一个答题区域并显示其属性。
- `移动区域`：拖动整个区域。
- `调整边界`：拖动四个边界点修改 `bbox`。
- `新增区域`：在原图上框选新的答题区域。
- `删除区域`：删除误识别区域，但必须二次确认。
- `修改题号`：修改区域显示题号。
- `选择题目`：从已保存的试题目录中选择对应题目，写入统一的 `question_id`。
- `预览裁剪`：按当前 `bbox` 显示实际裁剪结果，检查边界是否截断答案。
- `恢复自动结果`：放弃本次人工修改，恢复系统初始建议。
- `保存草稿`：保存当前区域调整，模板保持 `pending_review`。
- `提交复核`：校验区域数量、题号、`question_id`、坐标和重叠情况后提交。
- `发布模板`：只有复核通过后才能发布，调用 `POST /api/templates/{template_id}/publish`。
- `绑定考试`：将已发布模板绑定考试，调用 `POST /api/exams/{exam_id}/template`。

发布前必须阻止以下情况：

- 区域没有 `question_id`；
- `question_id` 与考试题目目录不一致；
- 区域存在重叠；
- 区域超出页面范围；
- 模板版本为空；
- 模板尚未发布；
- 考试已有学生提交后尝试替换模板。

### 4. 模板管理列表页

用于从考试管理页进入模板创建、查看和版本管理。页面应显示：

- 模板名称；
- 所属考试；
- 模板版本；
- 模板状态；
- 答题区域数量；
- 创建时间和发布时间。

按钮功能：

- `创建答题卡模板`：进入空白答题卡上传流程；
- `查看模板`：进入模板审查页；
- `继续编辑`：打开未发布模板草稿；
- `发布模板`：发布已完成复核的模板；
- `绑定考试`：将已发布版本绑定到考试；
- `创建新版本`：复制已发布模板生成新的可编辑版本；
- `查看历史版本`：查看旧版本，旧版本只读。

模板管理页的入口应放在考试详情页或教师工作台导航中，不能只放在批改复核页面里。推荐导航名称为“考试与答题卡模板”。

### 5. 学生答题卷识别结果页

采用“页面缩略图 + 当前答题区域 + 识别文本”的布局。该页面不重新划分区域，只使用已发布模板。

按钮功能：

- `开始识别`：提交学生答题卷，按考试绑定的模板处理。
- `切换页面`：查看不同扫描页。
- `切换题目`：按 `question_id` 查看对应答案区域。
- `查看原图`：查看学生答题卷的原始页面。
- `查看裁剪图`：查看实际送入手写OCR的区域图片。
- `查看模板对齐`：对比学生页面和空白模板的对齐结果。
- `标记空白`：显示模板差异检测判断，不允许前端直接伪造识别文本。
- `人工修正答案文本`：仅保存人工修正版本，保留原始OCR文本和修改记录。
- `标记需要复核`：将答案加入批改负责人的复核队列。
- `确认识别结果`：确认当前答案已可交给批改模块。
- `导出结果`：导出保持现有 `exam-document.v1` 输出结构的结果。

### 6. 批改复核工作台改造

保留现有 `GradingReview.vue` 的 AI 批改功能，增加识别模块产生的两个入口：

- `试题查看队列`：展示试题卷中的题目、题号和题目文本。
- `答案查看队列`：展示学生答案、题号、裁剪图片和OCR文本。

批改负责人继续负责 AI 评分建议、人工评分、批注和发布；识别模块只负责提供正确的题目和答案边界及文本。

## 三、前端调用顺序

```text
新建考试
  -> 上传试题卷
  -> 确认 questions[] 和 question_id
  -> 上传空白答题卡
  -> 自动生成模板草稿
  -> 教师调整答题区域
  -> 提交复核并发布模板
  -> 绑定考试
  -> 上传学生答题卷
  -> 固定区域裁剪和手写识别
  -> 将 answers[] 交给批改工作台
```

## 四、接口兼容要求

识别前端必须保持当前接口字段不变：

- `POST /api/process`：使用 `document_role=question_paper` 或 `answer_sheet`。
- `GET /api/submissions/{submission_id}`：查询已持久化的最终结果。
- `POST /api/templates/drafts`：创建空白答题卡模板草稿。
- `PUT /api/templates/{template_id}/review`：保存人工复核后的区域。
- `POST /api/templates/{template_id}/publish`：发布模板。
- `POST /api/exams/{exam_id}/template`：绑定模板。

前端不应把 `template_id` 直接作为学生答题卷的唯一依据。学生答题卷应传 `exam_id`，由后端解析考试当前绑定的模板版本。

## 五、当前版本单页面改造方案

### 5.1 改造范围

当前阶段只制作静态前端样式和页面内模拟交互，不连接后端，不调用识别接口，不修改现有接口和数据库。

当前版本不新增页面名称和业务路由，只改造现有教师端批改复核页面：

```text
frontend/src/views/teacher/GradingReview.vue
```

将原来的“原始答卷 / OCR文本”板块扩展为一个“识别处理中心”，内部使用四个标签：

```text
答题卡模板生成 | 试题卷输入 | 学生答题卷输入 | 答案输入
```

四个标签在同一页面内切换，不跳转到新的业务页面。

### 5.2 四个标签的实现边界

| 标签 | 当前版本状态 | 是否调用识别接口 | 主要作用 |
|---|---|---:|---|
| 答题卡模板生成 | 静态交互演示 | 否 | 模拟上传、区域生成、人工调整和发布状态 |
| 试题卷输入 | 静态交互演示 | 否 | 模拟上传试题卷和展示 `questions[]` |
| 学生答题卷输入 | 静态交互演示 | 否 | 模拟模板匹配、裁剪和展示 `answers[]` |
| 答案输入 | 只做静态样式 | 否 | 预留标准答案录入区域，后续交给批改模块 |

“答案输入”不传 `document_role=answer_key`，因为当前识别模块不支持标准答案OCR处理。当前只展示静态输入框、文件选择区域和评分规则占位信息，不实现提交、识别和保存逻辑。

## 六、四个静态版块的详细实现过程

当前版本的“交互”只改变页面本地状态和演示数据，不进行文件上传和后端请求。所有按钮都需要有可见的加载、成功、失败或禁用状态，但状态数据由前端模拟。

### 6.1 答题卡模板生成

#### 初始状态

标签显示：

- 考试选择框；
- 模板名称输入框；
- 模板版本输入框；
- 空白答题卡上传区域；
- `自动识别区域`按钮置灰，直到考试和文件信息完整。

#### 第一步：选择考试

前端加载考试列表，显示考试名称而不是只显示 `exam_id`。选中考试后显示：

- 考试名称；
- `exam_id`；
- 考试状态；
- 当前题目目录数量；
- 当前绑定模板名称和版本。

考试状态不允许绑定模板时，前端禁止后续发布和绑定操作，但最终校验仍由后端完成。

#### 第二步：上传空白答题卡

前端限制可上传的文件类型和大小，上传后显示：

- 文件名；
- 文件类型；
- 文件大小；
- 文件页数（后端返回后显示）；
- `自动识别区域`按钮。

静态阶段不调用接口。点击上传后，页面显示模拟文件名、文件类型、页数和“待生成区域”状态；点击 `自动识别区域` 后加载本地模拟区域数据。

#### 第三步：查看和调整区域

中心画布显示答题卡原图，并叠加自动识别的区域框。点击区域后，右侧面板显示：

- 页码；
- 题号；
- `question_id`；
- 区域类型；
- 归一化坐标；
- 自动识别来源；
- 是否需要复核。

交互规则：

- 拖动区域：整体移动矩形；
- 拖动边界点：调整矩形范围；
- `新增区域`：在画布上框选区域；
- `删除区域`：删除当前区域并要求二次确认；
- `绑定题目`：从考试题目目录选择对应的 `question_id`；
- `预览裁剪`：按当前坐标显示裁剪图片；
- `恢复自动结果`：撤销本次人工调整。

每次调整只更新前端草稿状态，不立即发布。

#### 第四步：保存和发布

点击 `保存模板草稿` 时只保存页面内的模拟草稿状态。前端在 `提交复核` 和 `发布模板` 前执行展示用校验：

- 所有区域都有 `question_id`；
- `question_id` 都存在于考试题目目录；
- 题号不重复；
- 区域坐标在 `0` 到 `1` 范围内；
- 区域没有明显重叠；
- 模板版本存在；
- 区域数量与题目目录一致。

通过后将模板状态从“待复核”切换为“已发布（演示）”，显示模板名称、版本和绑定考试，并锁定区域编辑按钮。刷新页面后不要求保留该模拟状态。

### 6.2 试题卷输入

#### 初始状态

标签显示：

- 考试选择框；
- 试题卷上传区域；
- `开始识别`按钮；
- 空状态提示。

#### 第一步：上传并模拟识别

静态阶段不调用接口。点击 `开始识别` 后使用本地模拟进度：

```text
文件上传 -> 页面解析 -> OCR识别 -> 题目切分 -> 跨页合并 -> 结果生成
```

识别期间禁用重复提交按钮，完成后展示预置的题目列表。

#### 第二步：查看结果

中心区域显示试题卷原图，右侧显示题目列表：

- 题号；
- 题干；
- 页码；
- 置信度；
- `question_id`；
- 是否需要复核。

点击题目后，原图跳转到对应页面并高亮识别区域。

#### 第三步：人工复核

按钮功能：

- `编辑题干`：修改最终题目文本；
- `修改题号`：修改题号并重新检查重复；
- `标记复核`：增加人工复核状态；
- `查看跨页内容`：查看题目合并前后的页面来源；
- `确认题目`：确认单题结果。

点击 `保存题目目录` 后，只在页面状态中保存最终题目结果。原始OCR文本和人工修正文本都用模拟数据展示，不写入后端。

### 6.3 学生答题卷输入

#### 初始状态

标签显示：

- 考试选择框；
- 自动显示考试绑定的模板名称和版本；
- 学生答题卷上传区域；
- `开始识别`按钮。

如果考试没有已发布模板，禁止开始识别，并提示先完成答题卡模板生成和绑定。

#### 第一步：上传和模拟处理

静态阶段不调用接口。页面根据选中的模拟考试显示一个模拟的已发布模板，并在点击 `开始识别` 后展示处理进度：

```text
文件上传 -> 页面匹配 -> 模板对齐 -> 固定区域裁剪 -> 空白检测 -> 手写OCR -> 结果保存
```

#### 第二步：查看答案结果

中心区域显示学生答题卡页面或当前题目裁剪图，右侧显示：

- 题号；
- `question_id`；
- 答案文本；
- 是否空白；
- OCR置信度；
- 对齐置信度；
- 风险标识；
- 是否需要复核。

按钮功能（全部只修改本地演示状态）：

- `查看原图`：显示学生答题卡原页面；
- `查看裁剪图`：显示实际送入手写OCR的图片；
- `查看模板对齐`：对比学生页面和空白模板；
- `人工修正答案`：修改展示文本并保留原始OCR文本；
- `标记需要复核`：加入批改负责人复核队列；
- `确认识别结果`：将结果状态改为可交给批改模块；
- `查看完整结果`：打开页面内的模拟完整结果抽屉。

### 6.4 答案输入静态版块

该标签当前只制作静态界面，不实现业务逻辑。页面包括：

- 标准答案文件选择框；
- 标准答案文本编辑区；
- 题号列表；
- 评分要点占位区；
- `导入答案`、`保存答案`、`交给批改模块`等置灰按钮；
- “当前版本暂不处理标准答案OCR”的提示。

不调用识别后端，不修改当前 `exam-document.v1` 输出接口。后续由批改模块负责人确定标准答案的保存格式和接口后，再单独接入。

## 七、页面内部状态管理

同一页面至少维护以下状态：

- `activePanel`：当前四个标签中的一个；
- `selectedExam`：当前考试；
- `templateDraft`：模板草稿和区域调整结果；
- `questionPaperResult`：试题卷识别结果；
- `answerSheetResult`：学生答题卷识别结果；
- `processingState`：上传、识别、成功、失败、待复核；
- `selectedQuestion`：当前选中的题目或答题区域；
- `reviewQueue`：待人工复核项。

切换标签时不清空其他模块结果，只切换当前工作视图。离开页面前，如果模板草稿或人工修正内容未保存，应弹出确认提示。

## 八、当前静态版本验收顺序

1. 页面名称和现有批改区域保持不变。
2. 四个标签都能在同一页面切换。
3. “答案输入”只显示静态样式，不发起请求。
4. 空白答题卡上传后能够展示模拟自动区域。
5. 教师能够在页面内调整区域、绑定模拟 `question_id`、保存模拟草稿。
6. 模板能够切换到“已发布（演示）”状态。
7. 试题卷能够展示模拟 `questions[]`。
8. 学生答题卷能够展示模拟 `answers[]`。
9. 答案输入能够展示静态录入样式，按钮保持禁用或提示未接入。
10. 识别结果能够在页面内切换到现有批改复核区域。
