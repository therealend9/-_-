# 文件处理模块真实文件测试报告

测试日期：2026-07-16

## 结论

本次验证使用仓库中已保存的真实文件，并实际运行了 `.venv-ocr` 中的 PaddleOCR；没有使用模拟 OCR 响应。

- 空白答题卡模板草稿：通过。真实两页 PDF 被渲染，自动提出 4 个书写区域，并按题目目录映射为 `q01` 至 `q04`。结果仍为 `pending_review`，符合“自动化后人工复核”的设计。
- 固定模板学生答题卷：链路通过但识别质量不通过。模板发布、考试绑定、页面对齐、区域裁剪、空白检测、结果持久化及幂等重试都执行成功；真实手写页的 37、38、39 三个答案没有获得可用文本，输出已正确标注 `needs_review=true`。
- 自由版式负向对照：通过。对真实学生作文错误地以 `question_paper` 角色提交，系统返回空 `questions[]`，未虚构题目。
- 自由版式真实试题卷提取：本次未形成能力结论。仓库内没有可确认的“大学思政课、纯主观题、正式试题卷”真实样本；公开检索到的候选试卷含客观题，因此未混入测试集。

当前模块可用于“空白答题卡生成候选模板 + 人工复核 + 结果带风险标识输出”。对于手机拍摄、透视变形、强阴影的真实手写答题卡，当前对齐与手写 OCR 尚不足以自动产出可直接使用的文本，必须进入人工复核队列。

## 运行环境

- Python：3.11.9
- OCR：PaddleOCR（项目 `.venv-ocr`）
- 全部真实文件运行耗时：约 86 秒，包含模型初始化与三条处理链路。
- 自动化回归：`27 passed in 1.19s`。

37、38、39 的 `question_id` 与考试题目目录、人工复核模板完全一致，说明 ID 对齐与输出契约有效。问题在于图像对齐置信度低和手写 OCR 结果不足，而不在输出格式。

## 样本边界

- T2 是公开的已填写答题卡，包含客观题区域。因此它只用于验证固定区域切分、手写检测、风险标记和结果结构，不作为“大学思政课纯主观题正式答题卡”的最终精度指标。
- T4 是负向对照，不证明自由试题卷能够成功切题。
- 曾检索到的公开马原试卷候选含客观题，违反当前产品只处理主观题的范围，未下载或纳入指标。

要给出自由版式试题卷切分的真实准确率，以及正式纯主观题答题卡的文本准确率，仍需要成对采集：同一考试的试题卷、空白答题卡模板和 20 份以上已匿名学生答题卡，并人工标注题号、区域坐标与标准转写文本。
