# 前端考试文件处理接入说明

## 页面与状态

前端有三个工作流：试题卷处理、答题卡模板制作、答题卡处理。

## 输出格式

`POST /api/process` 的成功响应固定遵循 `exam-document.v1`。顶层
`schema_version` 必为 `"exam-document.v1"`；接入方应先校验该字段，再根据
`document_role` 读取内容：

- `question_paper`：读取 `questions[]`。
- `answer_sheet`：读取 `answers[]`。

完整字段、示例和 JSON Schema 见 [考试文件识别 API 接口文档](考试文件识别API接口文档.md)。

### 试题卷处理

提交：`POST /api/process`，`document_role=question_paper`。自由版式试题卷可以不带
`exam_id`，结果读取 `questions[]`。但需要制作答题卡模板时，必须先创建考试并在此处
传入该考试的 `exam_id`。后端会将本次输出的 `questions[]` 保存为该考试唯一的题目目录。
目录中的 `question_id` 是后续模板与答题卡答案的唯一关联键。

### 答题卡模板制作

1. `POST /api/exams` 创建考试，前端展示 `exam_name`，后台使用稳定的 `exam_id`。
2. 使用这个 `exam_id` 上传试题卷；调用 `GET /api/exams/{exam_id}/questions` 确认题目目录。
3. `POST /api/templates/drafts` 上传空白答题卡，并传入同一个 `exam_id`，得到 `pending_review` 模板。
   后端按答题区域的页面阅读顺序映射题目目录；答题卡印刷题号保存在 `template_label`，不能作为最终关联键。
4. `GET /api/templates/{template_id}` 读取页面与候选区域。参考页通过
   `/api/template-assets/{template_id}/page_001.png` 访问。
5. 编辑器显示候选框。教师可拖动、删除、新增，确认过的区域将
   `needs_review` 改为 `false`。前端须保留后端给出的 `question_id`；新增区域时只能从
   题目目录中选择 `question_id`。同一题的多个答题区使用相同 `question_id`，但 `order` 必须不同。
6. `PUT /api/templates/{template_id}/review` 保存完整 `pages` 数组。
7. `POST /api/templates/{template_id}/publish` 发布。后端拒绝任何仍带
   `needs_review: true` 的区域。
8. `POST /api/exams/{exam_id}/template` 绑定已发布模板。后端会拒绝模板答案区域的
   `question_id` 集合与题目目录不一致的绑定请求。
9. 模板确认后调用 `PATCH /api/exams/{exam_id}/status`，请求体为
   `{"status":"published"}`。发布后的考试不能再变更模板；答题卡已有提交时，即使考试仍是
   `draft`，后端也会拒绝更换模板。

考试选择器应展示名称而不是只展示 ID。例如：

```text
显示：马原15-16期末试卷
值：  exam_id = 1
```

`GET /api/exams` 会返回 `exam_id`、`exam_name`、`template_id`、`template_name`、
`template_version` 和模板状态。前端提交时只发送选中项的 `exam_id`。

### 答题卡处理

提交：`POST /api/process`，`document_role=answer_sheet`，并附带 `exam_id`。前端
不提交 `template_id`。成功结果读取 `answers[]`；其中 `question_id` 与该考试的
`questions[]` 完全一致，不能用答题卡印刷题号自行关联。每条答案的 `needs_review` 和
`risk_flags` 决定是否显示人工复核入口。

## 接口失败处理

- `422 TEMPLATE_NOT_BOUND`：考试未绑定已发布模板。
- `422`：模板尚有未确认区域、坐标不合法或请求字段不合法。
- `503 OCR_DEPENDENCY_MISSING`：服务运行环境缺少手写/扫描 OCR 依赖。

启动前先调用 `GET /health`。当 `status=degraded` 时，前端可继续使用模板管理和
文本型 PDF 试题处理，但应禁用扫描件、模板草稿自动识别和答题卡 OCR 提交。
