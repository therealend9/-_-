# 真实公开答题卡样本测试报告

测试日期：2026-07-21  
测试环境：项目 `.venv-ocr`，当前本地 PaddleOCR/PaddleX 模型缓存，输出接口 `exam-document.v1`

## 1. 测试范围

本次只验证当前模块已有能力：

- 读取公开图片并完成文件解析；
- 从自由版式中识别印刷题号和题目片段；
- 对固定模板配置的身份区域进行单独裁剪和 OCR；
- 检查姓名、学号是否进入 `answers[]`；
- 检查失败时是否返回复核标识，而不是伪造身份信息。

本次不把公开网页图片当作“真实高校思政考试标准数据集”。公开搜索结果中混有高考、数学、考研和学习笔记图片，未能核验为大学思政期末正式答卷的样本已明确标记，不能据此宣称思政场景识别率。

样本文件、来源地址和逐份输出见：

- `test_artifacts/identity_fields_real/sample_manifest.json`
- `test_artifacts/identity_fields_real/results/`

## 2. 真实样本结果

### 样本一：填涂型答题卡

文件：`candidate_university_answer_sheet.jpg`  
来源页面：`http://dashangu.com/postimg_23318672.html`

页面上能看到姓名栏、准考证号、座位号和科目代码，但版面主体是填涂题。实际执行：

```text
scripts/run_full_processor.py candidate_university_answer_sheet.jpg
document_role=question_paper
```

真实输出为：

```json
{
  "schema_version": "exam-document.v1",
  "document_role": "question_paper",
  "layout_type": "free",
  "questions": []
}
```

结论：文件解析成功，但没有被识别为当前项目需要的主观题试题卷。它可作为“非目标版式”边界样本，不能用来证明主观题切分有效。

### 样本二：期末答题卡

文件：`candidate_final_exam_answer_sheet.png`  
原始来源页面：`http://www.renrendoc.com/paper/153240531.html`  
原始图片为 GIF，测试前转换为 PNG。

真实输出识别出 7 个题目片段，其中 6 个印刷题号片段置信度为 `0.89`，另有一个片段被标记：

```json
{
  "question_id": "8",
  "question_no": "8",
  "question_text": "8.[A] [B] [C] [D].126 1鐩存帴瀵绘暟锛?4锛?鍒楋細6鍒?",
  "needs_review": true,
  "risk_flags": ["contains_low_confidence_block"]
}
```

结论：当前模型能读取清晰的印刷题号和题面，但混合版式、题目文字和手写/噪声内容会产生乱码或低置信度，系统正确地要求复核。该样本不是思政卷，只能验证通用版面能力。

### 样本三：公开手写答卷

文件：`public_handwritten_answer_sheet_sample.png`  
文件含打印身份栏和较长手写答案。本次在模板中新增两个独立保护区域：

- `student_name`：身份裁剪区；
- `student_no`：身份裁剪区。

实际身份提取结果：

```json
{
  "status": "needs_review",
  "fields": {
    "student_name": {
      "value": "",
      "confidence": 0.0,
      "needs_review": true,
      "risk_flags": ["low_identity_ocr_confidence", "identity_value_empty"]
    },
    "student_no": {
      "value": "",
      "confidence": 0.0,
      "needs_review": true,
      "risk_flags": ["low_identity_ocr_confidence", "identity_value_empty"]
    }
  }
}
```

同一次运行的公共输出字段为：

```text
schema_version, submission_id, exam_id, exam_name,
document_role, layout_type, answers, template_id,
template_name, template_version
```

`answers[]` 中没有 `student_name`、`student_no`、`姓名` 或 `学号` 字段，也没有把身份 OCR 结果混进答案文本。身份结果只写入受保护的内部绑定记录流程，并因置信度为 0 被要求人工复核。

## 3. 结论

当前版本的真实测试结论是：

1. 文件接收、图片解析和 `exam-document.v1` 输出可以正常运行。
2. 清晰印刷题号的识别基本正常；复杂或混合版式会产生低置信度结果，并能标记复核。
3. 身份区域可以从答题区单独裁剪，身份字段不会进入公共 `answers[]`。
4. 对当前公开手写样本，姓名和学号没有被可靠识别出来，系统没有猜测，而是返回空值并要求复核。这说明“隔离脱敏”流程有效，但“手写姓名/学号自动识别”仍未达到可无人值守的可靠程度。
5. 本次没有找到足以证明“大学思政课正式主观题答题卡识别率”的公开、可核验样本集。后续验收应使用经学校或教师授权的脱敏样本，并记录人工真值。
