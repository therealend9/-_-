# 考试文件识别模块剩余问题

## 范围

本文只记录会影响当前后端识别模块正确性、可靠性或上线判断的问题。

不包含以下事项：前端页面和交互、数据库替换、异步任务、监控、部署治理，以及纯性能或体验类优化。本模块当前只输出试题 `questions[]` 或答题卡答案 `answers[]`，不负责自动评分。

## 已解决的关键约束

- 试题卷、答题卡模板与最终答案共用同一套 `question_id`。
- 模板绑定时校验考试存在、考试状态、模板发布状态、明确模板版本及题目 ID 集合。
- 答题卡已有成功提交后，不能替换为另一套模板。
- 未复核区域不能发布模板。

## 当前仍需改进的问题

### P0-01 真实试题卷与答题卡的端到端验收

当前自由格式试题卷只用 15-16 马原的文本型 PDF 验证过；固定答题卡只完成了合成样本和结构链路验证。真实扫描件、拍照件和手写中文答案尚未形成可量化结论。

影响：无法判断题目切分、页面对齐、区域裁剪、空白判断和最终答案文本在真实考试场景中的可靠性。

需要样本：

- 同一套试题卷的文本 PDF、扫描 PDF 和手机照片。
- 空白答题卡及多个学生填写版本。
- 正常书写、分点作答、涂改、空白作答、低清晰度、倾斜和跨页作答样本。

验收内容：

- 试题卷题目数、题号和题干边界正确。
- 答题卡页面能对齐到模板，区域不串题、不截断。
- `question_id` 与题目目录逐条一致。
- 空白区域、低置信度和对齐失败能正确标记 `needs_review`。

状态：`待真实样本`

### P0-02 手写中文 OCR 的可用性尚未评估

当前 `ocr_mode=handwriting` 已进入固定区域流程，但底层仍使用当前 PaddleOCR 路线；它是否适合学生中文手写、分点答案和涂改答案尚无真实数据支撑。

影响：即使区域定位正确，最终 `answers[].answer_text` 也可能无法供后续业务使用。

处理方向：基于 P0-01 的真实样本量化字错率和整题可读率，再决定是否需要专用手写模型、区域放大、图像增强或行级识别。

状态：`待真实样本`

### P0-03 已发布模板仍可被同 ID 修改

`review_template(...)` 目前可以读取已发布模板并重新保存页面区域。因此虽然考试发布或已有答题卡提交后不能改绑到其他模板，已绑定模板本身仍可能被原地修改。

影响：同一个 `template_id + version` 的区域坐标可能发生变化，导致历史提交和后续提交不再基于同一版模板。

期望规则：

```text
draft -> pending_review -> published
published -> 禁止修改
需要修改 -> 新建 template_id 或新建 version，再重新走复核和绑定
```

状态：`待实现`

### P0-04 绑定模板后题目目录尚未完全冻结

当前绑定模板后会校验题目目录与模板的 `question_id` 集合一致，但重新上传试题卷时，如果新旧题目使用相同 ID、题干却不同，题目目录仍会被覆盖。

影响：模板和答案的 ID 仍然一致，但 `questions[]` 的题干可能已不是该答题卡对应的试题，造成语义错配。

期望规则：模板绑定后冻结题目目录的版本和内容哈希；若要更换试题卷，必须创建新的考试或显式走新版本流程。

状态：`待实现`

### P1-01 自动模板区域映射只支持一对一候选框

自动建模按页面阅读顺序，把一个候选答题框映射到一个题目目录项。区域数量与题目数量不一致时会标记 `question_count_mismatch` 并要求人工复核，但不会自动表达“一题多个书写区”或“一个书写区覆盖多问”的关系。

影响：复杂答题卡仍可人工配置，但自动建模结果不能直接作为可靠起点。

处理方向：在真实答题卡样本中确认是否存在这类布局；若存在，增加区域分组与 `question_id` 复用的自动建议，并保留人工确认。

状态：`待真实模板确认`

## 建议处理顺序

```text
1. 收集真实试题卷、空白答题卡和学生答题卡样本
2. 完成 P0-01 的端到端验收，并量化 P0-02 的手写 OCR 质量
3. 禁止已发布模板原地修改
4. 冻结已绑定考试的题目目录内容
5. 根据真实答题卡决定是否增强多区域自动映射
```
