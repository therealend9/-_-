# 考试文件识别模块工作流接入指南

## 1. 模块边界

工作流只需要接入三种场景：

| 场景 | 输入 | 输出 |
|---|---|---|
| 试题卷 | 自由版式试题 PDF/图片/DOCX | `questions[]` |
| 空白答题卡 | 空白模板文件 | 模板草稿、复核后模板 |
| 学生答题卡 | 已绑定模板的学生答题卡 | `answers[]` |

模块不处理独立标准答案、不保存 `answer_keys[]`、不评分、不做答案比对。

## 2. 推荐流程

```text
创建考试 draft
  -> 上传试题卷并保存 questions[]
  -> 上传空白答题卡生成模板草稿
  -> 前端展示候选区域，老师人工复核
  -> 发布模板并绑定考试
  -> 发布考试
  -> 上传学生答题卡并读取 answers[]
```

试题卷识别使用 `document_role=question_paper`；学生答题卡使用 `document_role=answer_sheet` 和 `exam_id`。空白答题卡不调用 `/api/process`，而是调用 `/api/templates/drafts` 生成固定模板。

## 3. 关键约束

- `questions[].question_id`、模板区域 `question_id`、`answers[].question_id` 必须完全一致。
- 学生答题卡不传 `template_id`，服务端按 `exam_id` 查询已绑定模板。
- 模板必须发布且版本明确，考试状态必须允许绑定。
- 已有学生提交后不能更换考试模板。
- `needs_review=true` 的试题或答题卡结果应进入人工复核，不应直接进入评分流程。

完整字段和示例见 [考试文件识别 API 接口文档](考试文件识别API接口文档.md)。
