# 思政 Agent 项目优化工作流

本文档用于把项目从“演示型系统”逐步优化为“可真实试点运行的教学平台”。建议按阶段推进，每完成一项就补充实际链接、接口、截图、测试记录或验收说明。

> 2026-07-11 更新：知识图谱、DeepSeek 问答和 AI 备课已完成真实服务接入；后续重点为账号治理、课程数据、自动化测试与部署。

## 优化原则

- 优先打通真实业务闭环，再扩展展示页面。
- 优先完成账号、课程、任务、反馈、知识库这些基础能力。
- AI 能力必须可解释、可追溯、可审核。
- 每个阶段都要有可验证的交付物，避免只停留在界面演示。

## 阶段 0：现状梳理与基线确认

目标：确认当前项目能运行、能登录、能访问主要页面，形成后续优化的基准。

任务清单：

- [ ] 梳理前端页面清单，标记“真实接口 / Mock 数据 / 静态展示”状态。
- [ ] 梳理后端接口清单，标记“已实现 / 半实现 / 缺失 / 仅演示数据”状态。
- [ ] 梳理数据库表结构，确认用户、课程、班级、作业、测验、知识库、审计日志等核心表。
- [ ] 修复明显编码异常、中文乱码、文案缺失问题。
- [ ] 确认本地启动方式，补充前端、后端、数据库启动说明。
- [ ] 记录当前可用演示账号和权限范围。

验收标准：

- [ ] 可以本地启动前端和后端。
- [ ] 学生、教师、管理员三类演示账号可以登录。
- [ ] 有一份明确的“已完成 / 待完善”功能清单。

## 阶段 1：账号与组织体系

目标：让系统具备真实用户管理能力，而不是只依赖演示账号。

任务清单：

- [x] 新增用户注册或管理员创建用户接口。
- [ ] 新增管理端“新增用户”表单，并接入真实后端接口。
- [ ] 完善用户字段：账号、姓名、手机号/邮箱、角色、学院、班级、状态。
- [ ] 完善密码哈希、密码强度、重复账号校验。
- [ ] 增加密码重置、初始密码修改、忘记密码流程。
- [ ] 完善用户启用、禁用、角色变更、权限范围变更。
- [ ] 将批量导入从固定演示数据改为真实 Excel 上传解析。
- [ ] 建立学院、专业、班级、课程、用户之间的绑定关系。
- [ ] 预留统一身份认证接口。

验收标准：

- [ ] 管理员可以创建学生、教师、管理员账号。
- [ ] 新账号可以登录并进入对应角色页面。
- [ ] 禁用账号无法登录。
- [ ] 批量导入可以从真实文件生成用户数据。

## 阶段 2：课程与教学基础数据

目标：建立真实课程空间，为后续任务、测验、AI 教学提供数据基础。

任务清单：

- [ ] 完善课程创建、编辑、归档接口。
- [ ] 完善教学班创建、教师绑定、学生选课关系。
- [ ] 支持课程章节、知识点、教学目标、课程资源维护。
- [ ] 管理端课程管理页面接入真实接口。
- [ ] 教师端只展示自己授课的课程和班级。
- [ ] 学生端只展示自己已选课程和学习任务。

验收标准：

- [ ] 管理员可以创建课程和教学班。
- [ ] 教师可以看到自己的课程。
- [ ] 学生可以看到自己加入的课程。
- [ ] 课程章节和知识点能被后续任务引用。

## 阶段 3：学习任务、测验与作业闭环

目标：打通“课程内容 -> 学习任务 -> 测验/作业 -> 批改反馈 -> 学习报告”的核心闭环。

任务清单：

- [ ] 教师可发布课前导学任务。
- [ ] 教师可发布课堂测验和课后作业。
- [ ] 学生可提交测验答案和作业。
- [ ] 支持客观题自动判分。
- [ ] 支持主观题教师批改、AI 辅助评分、人工复核。
- [ ] 支持评分量规 Rubric 与作业绑定。
- [ ] 支持作业退回、重新提交、批注反馈。
- [ ] 学生端展示个人成绩、反馈和改进建议。
- [ ] 教师端展示班级完成率、平均分、薄弱点。

验收标准：

- [ ] 教师可以完整发布一次任务。
- [ ] 学生可以完成并提交。
- [ ] 教师可以查看、批改和反馈。
- [ ] 学生可以看到结果和反馈。

## 阶段 4：知识库与可信溯源

目标：让 AI 回答、教师备课、学生学习都基于可信来源。

任务清单：

- [ ] 支持教材、课件、政策文件、案例材料上传。
- [ ] 支持资源审核状态：待审核、已通过、已驳回。
- [ ] 支持资源分段、索引、向量化。
- [ ] 支持按课程、章节、知识点绑定资源。
- [ ] AI 回答必须返回引用来源。
- [ ] 来源追踪页面展示命中片段、文件、章节、审核状态。
- [ ] 支持资源版本管理和失效处理。
- [ ] 管理端可重新索引、下线、审核知识资源。

验收标准：

- [ ] 上传资源可以被检索。
- [ ] AI 回答能展示来源。
- [ ] 未审核资源不能进入正式问答。
- [ ] 管理员可以追踪某次回答引用了哪些材料。

## 阶段 5：AI 教学工作流

目标：把 AI 从简单问答升级为教师和学生的工作流助手。

任务清单：

- [ ] 学生端 AI 问答支持追问、提示引导、来源引用。
- [ ] 学生端避免直接给出作业答案，改为启发式引导。
- [ ] 教师端支持 AI 生成教案、课堂问题、小测题、讨论题。
- [ ] 教师端支持根据班级学情生成教学建议。
- [ ] 教师端支持 AI 辅助评分和 Rubric 对齐解释。
- [ ] 管理端支持 AI 内容审核、风险提示、价值导向检查。
- [ ] 记录 AI 调用日志、输入输出、引用来源、审核结果。
- [ ] 支持模型配置、提示词模板、知识库范围配置。

验收标准：

- [ ] 教师可以用 AI 生成一份可编辑教案。
- [ ] 学生问答有明确来源和引导逻辑。
- [ ] AI 生成内容进入审核流程后才能正式发布。
- [ ] 管理员能查看 AI 调用和审核日志。

## 阶段 6：学习分析与预警

目标：为教师和管理员提供可行动的数据分析，而不是只展示统计数字。

任务清单：

- [ ] 建立学习行为数据采集：登录、访问、提交、测验、互动、问答。
- [ ] 学生端展示个人学习进度、薄弱知识点、建议任务。
- [ ] 教师端展示班级完成率、知识点掌握度、学生风险预警。
- [ ] 管理端展示课程运行、用户活跃、资源使用、AI 调用统计。
- [ ] 支持低参与、低成绩、逾期未交、频繁错误等预警规则。
- [ ] 支持教师干预记录和后续效果追踪。

验收标准：

- [ ] 教师能识别需要干预的学生。
- [ ] 学生能看到明确的下一步学习建议。
- [ ] 管理员能看到课程和平台运行状态。

## 阶段 7：安全、合规与审计

目标：满足真实教学系统上线所需的基础安全与合规要求。

任务清单：

- [ ] 完善接口鉴权和角色权限校验。
- [ ] 避免越权访问其他用户、班级、课程数据。
- [ ] 增加审计日志：登录、创建、修改、删除、审核、导入、AI 调用。
- [ ] 敏感数据脱敏展示。
- [ ] 增加登录失败限制、Token 过期、退出登录。
- [ ] 增加文件上传类型、大小、安全校验。
- [ ] 增加 AI 内容安全、敏感词、价值导向风险检测。
- [ ] 完善隐私声明、数据使用说明、操作留痕说明。

验收标准：

- [ ] 不同角色不能访问越权接口。
- [ ] 关键操作有审计记录。
- [ ] 禁用用户和过期 Token 不能继续调用接口。
- [ ] 上传文件和 AI 输出有基础安全校验。

## 阶段 8：工程化与部署

目标：让项目可以稳定交付、部署、演示和迭代。

任务清单：

- [ ] 整理环境变量和 `.env.example`。
- [ ] 增加数据库迁移脚本和种子数据。
- [ ] 增加前后端启动脚本说明。
- [ ] 增加接口测试、核心业务单元测试、端到端测试。
- [ ] 增加日志、错误码、统一异常处理。
- [ ] 增加 Docker 或一键部署脚本。
- [ ] 区分开发、测试、演示、生产配置。
- [ ] 清理无效 Mock 数据和重复旧目录。
- [ ] 补充 README、接口文档、验收说明。

验收标准：

- [ ] 新环境可以按文档完成部署。
- [ ] 核心接口测试通过。
- [ ] 前端构建通过。
- [ ] 数据库初始化后可以直接登录并体验核心流程。

## 推荐优先级

P0：必须优先完成，决定项目是否能真实试点。

- [ ] 账号与组织体系
- [ ] 真实课程和班级数据
- [ ] 作业、测验、批改反馈闭环
- [ ] 知识库上传、审核、检索、溯源
- [ ] AI 问答引用来源

P1：增强教学价值和管理价值。

- [ ] AI 教案、小测、评分辅助
- [ ] 学习分析与风险预警
- [ ] 管理端审计日志
- [ ] 批量导入真实 Excel
- [ ] 权限范围与越权校验

P2：提升产品成熟度。

- [ ] 统一身份认证
- [ ] 消息通知
- [ ] 移动端适配
- [ ] 第三方教务系统集成
- [ ] Docker 部署和自动化测试
