## 6. 接口与 mock 现状

当前已有接口封装：

- `src/api/student.js`：学生工作台、课前学习、导学详情、问题提交、预习小测、AI问答、反馈详情、批改详情、理论溯源
- `src/api/teacher.js`：教师工作台、教学设计、发布教案、课前学情
- `src/api/admin.js`：用户、课程、知识库、AI审核、量规、作业、日志、系统设置、角色权限、组织架构
- `src/api/http.js`：通过 `VITE_USE_MOCK` 和 `VITE_API_BASE_URL` 切换 mock/真实接口

仍直接使用本地常量或 `mock/data.js` 的区域：

- 三个 Layout 顶栏/提醒数据
- 教师课中/课后页面：课堂互动、随堂出题、批改复核、班级报告、资源库
- 学生通知、个人中心、学习报告
- 三端首页部分状态卡和提醒文案

接口层下一步原则：

- 页面组件只调用 `src/api/*.js`
- mock 数据只留在 API 层或 mock 层
- 真实接口路径只在 `src/api/*.js` 中维护
- `VITE_USE_MOCK=false` 时页面不应直接依赖本地静态数组
